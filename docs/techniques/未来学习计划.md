# NestJS Techniques 学习计划

基于官方 [Techniques](https://docs.nestjs.com/techniques) 目录，按 **实际业务后端开发**（本仓库：REST + JWT + TypeORM + Redis）安排优先级。侧边栏是文档目录，**不是**学习顺序。

读完本文，你应能回答：

- 哪些 Techniques 必须先掌握？哪些可以按需再学？
- 本仓库已经覆盖了哪些主题？
- 下一步最值得补哪几块？

> 官方入口：[docs.nestjs.com/techniques](https://docs.nestjs.com/techniques)

### Caching 补充说明

官方 `CacheModule`（`@nestjs/cache-manager`）有两种用法：

| 用法                                    | 适用                           | 注意                                          |
| --------------------------------------- | ------------------------------ | --------------------------------------------- |
| 注入 `CACHE_MANAGER` 手动 `get/set/del` | 业务数据（权限、配置、按用户） | 与本仓库自建 Redis 思路一致                   |
| `CacheInterceptor` 自动缓存 HTTP 响应   | 公开、无用户差异的 **GET**     | 仅 GET；不能用 `@Res()`；**GraphQL 下不可用** |

官方 Warning：GraphQL 中拦截器按 field resolver 分别执行，`CacheModule` 的响应缓存不能正常工作。本仓库是 REST，且未用官方拦截器缓存，**可忽略该警告**。

权限这类「按 userId 失效、写时删缓存」的场景，Service 手动读写往往比全局 `CacheInterceptor` 更合适。

## 5. 决策速查

```
要不要现在学这个 Techniques？
├─ 配置 / 数据库 / 校验 / 日志 / 缓存 → 必学（P0），本仓库大多已落地
├─ 序列化 / 上传 / 定时 / 队列 / 事件 / 调外部 HTTP / API 版本 → 有需求就学（P1）
└─ Cookie/Session、压缩、流式下载、SSE、Fastify、Mongo、MVC → 场景驱动（P2）
```
