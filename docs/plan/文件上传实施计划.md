# 用户头像文件上传实施计划

## 方案边界

- 学习阶段使用“存储接口 + 本地磁盘实现”，文件经 Nest 静态路由访问。
- 正式开发时增加阿里云 OSS 实现，通过配置切换，不改 Controller 和头像业务。
- `POST /file/upload/avatar` 只服务当前登录用户，multipart 字段名为 `avatar`；保存成功后直接更新 `users.avatar`。
- 首版只允许 JPEG、PNG、WebP，默认最大 2 MiB；不实现大文件、分片、断点续传、图片裁剪、通用附件表及 `File` 实体。

## 实施步骤

1. 新增独立 storage 配置域，配置存储驱动、本地目录、公开 URL、大小限制和 MIME 白名单。
2. 在 File 模块定义统一存储契约，并实现安全命名、落盘、删除和 URL/key 转换能力。
3. 完成头像上传、当前用户 URL 回写、失败回滚和旧头像清理。
4. 提供 `/uploads` 静态访问，补充 JWT、接口限流、文件签名校验和 Swagger multipart 契约。
5. 增加本地存储、头像业务和 HTTP 上传测试，执行 lint、测试及构建验证。

## 环境变量

```dotenv
STORAGE_DRIVER=local
UPLOAD_DIR=uploads
UPLOAD_PUBLIC_BASE_URL=http://localhost:4000/uploads
UPLOAD_MAX_SIZE_BYTES=2097152
UPLOAD_ALLOWED_MIME_TYPES=image/jpeg,image/png,image/webp
```

`UPLOAD_DIR` 的相对路径基于后端进程工作目录。部署时应挂载持久卷，并让 `UPLOAD_PUBLIC_BASE_URL` 与反向代理或 CDN 的实际公开地址一致。

## 后续阿里云 OSS 演进

- 新增实现相同契约的 `AliyunOssStorageProvider`，通过 `STORAGE_DRIVER=oss` 切换。
- 增加 endpoint、bucket、region、公开域名及凭证配置；凭证只通过部署密钥注入。
- 生产优先使用 RAM 最小权限或实例角色，不将 AccessKey 写入仓库。
- 公共对象可保存稳定 CDN URL；私有对象应在数据库保存 object key，访问时生成短期签名 URL，不能保存会过期的签名 URL。
